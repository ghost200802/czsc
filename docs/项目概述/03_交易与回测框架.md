# 交易与回测框架

本模块提供策略执行、回测引擎、绩效分析和权重管理功能，是 CZSC 框架从信号到交易的桥梁。代码位于 `czsc/traders/` 目录。

## 模块概述

交易与回测框架实现了完整的策略生命周期管理：信号配置 → 信号计算 → 事件触发 → 仓位管理 → 回测执行 → 绩效评估。

## 核心组件

### CzscSignals — 信号计算引擎

定义于 `czsc/traders/base.py`，负责根据信号配置自动计算信号序列。

主要功能：
- 接收标准化K线数据和信号配置列表，批量计算信号
- 支持多周期信号联合计算
- 输出包含时间戳、品种、信号key/value 的完整信号序列

辅助函数包括 `generate_czsc_signals`（生成信号）、`check_signals_acc`（验证信号准确率）、`get_unique_signals`（获取去重信号列表）。

### CzscTrader — 交易执行器

定义于 `czsc/traders/base.py`，是策略的核心执行单元。

主要功能：
- 管理多个 Position（仓位）对象
- 根据信号和事件自动触发交易操作
- 维护交易状态和持仓信息
- 支持多策略组合运行

### Position — 仓位管理

定义于 `czsc/py/objects.py`，表示一个完整的交易策略单元。

结构组成：
- **opens**：开仓事件列表（定义开多/开空条件）
- **exits**：平仓事件列表（定义平多/平空条件）
- **operate**：当前操作状态（开多/平多/开空/平空/持有）

### DummyBacktest — 虚拟回测

定义于 `czsc/traders/dummy.py`，提供基于买卖信号的简单回测功能。

### WeightBacktest — 权重回测

由 Rust 扩展 `rs_czsc` 提供，是项目推荐的回测引擎。

核心思想：权重驱动（非买卖信号），权重范围 [-1, 1]，正权重表示多头，负权重表示空头，连续调整持仓。

核心流程：
1. **品种日收益计算**（`get_symbol_daily`）：根据权重变化和价格变动计算每日策略收益，扣除交易成本
2. **开平交易记录生成**（`get_symbol_pairs`）：将权重信号序列转换为开平仓交易对，使用栈结构 FIFO 配对
3. **绩效聚合**：汇总所有品种的日收益，计算组合级绩效指标

支持时序策略（各品种等权平均）和截面策略（各品种直接相加）两种模式。

### PairsPerformance — 交易对绩效分析

定义于 `czsc/traders/performance.py`，用于分析单次开平仓交易的盈亏情况，包括交易胜率、单笔收益（BP）、持仓天数等指标。

辅助函数包括 `combine_dates_and_pairs`（合并日级别和交易对级别数据）、`combine_holds_and_pairs`（合并持仓和交易数据）。

## 策略优化

定义于 `czsc/traders/optimize.py`，提供基于 Optuna 的策略参数优化功能。

- **OpensOptimize**：开仓参数优化
- **ExitsOptimize**：平仓参数优化

## 权重管理客户端

### CzscWeightsClient（CWC）

定义于 `czsc/traders/cwc.py`，基于 ClickHouse 数据库的策略持仓权重管理客户端。

功能覆盖：数据库表初始化、策略元数据 CRUD、持仓权重发布与查询、策略收益发布与查询、策略生命周期管理。

详见 `docs/instructions/cwc_usage_guide.md`。

### RedisWeightsClient（RWC）

定义于 `czsc/traders/rwc.py`，基于 Redis 的轻量级权重管理客户端。

### 权重回测工具

定义于 `czsc/traders/weight_backtest.py`，提供 `get_ensemble_weight`（集成权重计算）和 `stoploss_by_direction`（方向性止损）等辅助函数。

## 信号解析

定义于 `czsc/traders/sig_parse.py`，提供信号配置的解析与管理工具。

- **SignalsParser**：解析信号配置字符串，提取信号名称和参数
- `get_signals_config`：获取信号配置列表
- `get_signals_freqs`：获取信号涉及的K线周期

## 策略基类

定义于 `czsc/strategies.py`，提供策略开发的模板类。

- **CzscStrategyBase**：策略基类，封装常用策略开发方法，简化策略编写流程
- **CzscJsonStrategy**：基于 JSON 配置的策略类，支持通过配置文件定义策略
