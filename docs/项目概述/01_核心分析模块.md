# 核心分析模块

本模块是 CZSC 框架的基础，负责缠论数据结构定义、K线合成与分析引擎的实现。代码分布在 `czsc/core.py`、`czsc/py/` 目录以及 Rust 扩展 `rs_czsc` 中。

## 整体架构

核心分析采用 Python + Rust 双层实现：

- **Rust 层**（`rs_czsc`）：高性能实现缠论分析、K线合成、数据对象等，是默认执行路径
- **Python 层**（`czsc/py/`）：历史功能代码存档，提供完整参考实现，可通过环境变量 `CZSC_USE_PYTHON=1` 强制启用
- **混合入口**（`czsc/core.py`）：统一接口层，自动选择 Rust 或 Python 实现

## 数据对象体系

定义于 `czsc/py/objects.py` 和 `czsc/core.py`，构成缠论分析的完整数据模型。

### 基础K线对象

| 对象 | 说明 | 关键属性 |
|------|------|---------|
| RawBar | 原始K线数据 | symbol, dt, open, close, high, low, vol, amount, freq |
| NewBar | 经过包含关系处理后的K线 | 继承 RawBar，增加包含关系标记 |

### 缠论核心对象

| 对象 | 说明 | 关键属性 |
|------|------|---------|
| FX | 分型 | elements（组成K线）, fx_type（顶/底分型）, strong（强弱分型） |
| BI | 笔 | direction（方向）, fx_a/fx_b（起止分型）, bars（包含K线列表）, power（力度） |
| ZS | 中枢 | bis（组成笔）, zz（中枢区间）, direction（方向） |
| FakeBI | 虚笔 | 用于笔的延伸判断 |

### 交易相关对象

| 对象 | 说明 | 关键属性 |
|------|------|---------|
| Signal | 信号 | key（信号标识）, value（信号值）, score（评分） |
| Event | 事件 | signals_all/any/not（信号逻辑组合）, operate（操作指令） |
| Position | 仓位 | name, symbol, opens（开仓事件列表）, exits（平仓事件列表）, operate（当前操作） |

### 枚举类型

定义于 `czsc/py/enum.py`，包含 Direction（方向：多/空）、Freq（K线周期）、Mark（标记类型）、Operate（操作类型：开多/平多/开空/平空/持有）。

## CZSC 分析引擎

核心类定义于 `czsc/py/analyze.py`，是缠论分析的主要入口。

### 核心功能

- **K线包含处理**（`remove_include`）：对相邻K线进行包含关系判定与合并，遵循缠论包含关系处理规则（上升趋势取高高、下降趋势取低低）
- **分型识别**（`check_fx`）：在处理后的K线序列上识别顶分型和底分型
- **笔识别**（`check_bi`）：基于分型序列识别笔，包括新笔的延伸判断和旧笔的修正
- **中枢识别**：自动计算笔序列中的中枢
- **多级别联合分析**：支持同一品种不同时间周期的嵌套分析

### 使用方式

通过统一接口 `from czsc import CZSC` 导入，内部自动选择 Rust 或 Python 实现。分析引擎接收标准化K线序列，输出完整的缠论结构（分型、笔、中枢）。

## K线合成器

定义于 `czsc/py/bar_generator.py`，负责将低周期K线合成为高周期K线。

### 功能

- 支持从 Tick 级别到年线级别的全周期合成
- 提供标准K线格式化函数 `format_standard_kline`
- 提供交易时间判定函数 `is_trading_time`
- 提供周期结束时间计算函数 `freq_end_time`

### 关键文件

- `czsc/py/bar_generator.py` — BarGenerator 类及相关函数
- `czsc/core.py` — 统一导出接口

## 交易日历

定义于 `czsc/py/calendar.py`，提供中国A股市场的交易日历查询。

### 功能

- 判断指定日期是否为交易日（`is_trading_date`）
- 获取下一个/上一个交易日（`next_trading_date`、`prev_trading_date`）
- 获取指定范围内的所有交易日（`get_trading_dates`）
- 日历数据存储在 `czsc/py/china_calendar.feather` 中

## 环境变量管理

定义于 `czsc/envs.py`，集中管理项目运行时环境变量配置，包括 CZSC 日志级别、Rust/Python 选择等。
